Colour Schemes and Visual Variables

An example colour scheme and its description using Visual Variables ...

1. ColorBrewer

Let's consider a ColorBrewer sequential scheme, that works well for numeric or ordinal data ... data with a sequence.

The numbers here are the Red, Green and Blue components of the colour, all on a scale of 0-255. Colours are often defined in this way. But we need to define and describe them using the differentiable visual variables the Hue, Saturation and Lightness that combine to uniquely describe any colour, all colours.

2. Color Picker

We can find the Hue, Saturation and Lightness of the colours in an image with various tools, apps, websites and plugins.

I suggest the PineTools : Online Image Color Picker where you can select an image for analysis by browsing your file space or dragging and dropping an image into the web browser.

This allows you to analyse screen dumps. You can also specify images by URL and check them that way.

HUE

Remember that Hue is usually a number between 0 and 360. We can represent these as angles round a colour wheel, from 0 at the top (a red hue) round the rainbow and back to 359.

Here we add the numbers that represent the various hues - from 0-360.

Let's think about a particular colour scheme that might be used to show data and how hue varies within it - the ColorBrewer YlOrRd seven class scheme that we introduced above.
We can show the hues used in the colours as angles and positions on the colour wheel.

So 347 is just 13 / 360 Hue increments, or degrees, different to zero! So, quite close, quite similar - just 4% of the way around the colour wheel from 0.

What this shows is a gentle hue progression, in sequence from Yellow to Orange to Red in the seven colours used in the scheme ... hence the name of the colour scheme.

So Hue does vary a little in this scheme. It does so sequentially, so that we can detect the order.

LIGHTNESS

We can think of value and lightness and luminance and brightness as the same thing at this stage. You will see HSL and HSV and HSB descriptions of colour, and they do vary. But for now, let's consider the lightness numbers that I recorded above.

How do they vary across the scheme?

Well, let's create a graph!
The colours in the ColorBrewer YlOrRd seven class scheme go from left (1, light yellow) to right (7, dark red) here.

We can see that value or brightness or lightness decreases in order across the scheme, from mid 80s to mid 30s.

So lightness and hue both vary consistently and sequentially in this scheme to give it a visual order.
Those are two consistent encodings used in a single scheme, and probably both used in any graphic in which it is employed to encode the same data channel - a double encoding.

Let's try to show lightness on the colour wheel that we used to show the variations in hue.

We can represent, or encode this in our colour wheel with position. Here the darker colours are closer to the centre of the wheel.

Here the hues are shown on a colour wheel with seven rings that show the seven hues used with seven different levels of lightness.

If we add an annotation, linking the hues in the order in which they are used in the colour scheme, then we can see the order of the hues quite clearly. There is a sequence, a visual logic.

SATURATION - HSL

How about saturation - the richness or chroma of a colour that varies from high values that are highly saturated to low values that are pastel and washed out.

Well the graph 👆 and table above show little consistent variation in saturation as defined through HSL. When described using HSL, these are fully saturated colours with just one (slightly strange, seemingly inconsistent) variation in the colour used for class 6.

So it doesn't look as though saturation is being used to encode here - it hardly varies.

Here is the ColorBrewer YlOrRd seven class scheme.

The top row defines colours with the original values for Hue, Lightness and Saturation.

The bottom row shows the same Hue, and Lightness as in the original, but keeps Saturation constant at 98% under the HSL model. The only difference that I can detect is for class 6 where the dark red hue seems to stand out. Look - the sequence from class 5 to class 7 does not seem constant. The colour for class 6 is a bit full on!

This probably explains why cartographers, like Cindy Brewer, who is a Professor of Geography and Penn State University adjust colour schemes so that they vary in ways that we perceive to be continuous, even if this results in some discontinuities in the way that the visual variables of colour are applied.

Just to show what variation in saturation looks like, here is a graphic showing the original scheme again, in the middle row, and two schemes that vary only in saturation under HSL.

The two rows either side of this take the first (top) and final (bottom) colour in the sequence and vary the saturation consistently in the HSL colour description while keeping hue and lightness constant across the row of colours.

In the top row, saturation changes from a maximum in class 7 on the right, through to a minimum in class 1 to the left.

In the bottom row, saturation changes from a maximum in class 1 on the left, through to a minimum in class 7 to the right.

This should help you see the effects of varying the saturation of a single colour under HSL.

SATURATION - HSV

If we consider the HSV model, we see more variation in saturation than is revealed by HSL and less variation in lightness - but both vary consistently.

The following table shows the numbers returned from the PineTools : Online Image Color Picker for the same ColorBrewer sequential scheme, with the HSV numbers that are reported and considered above. But in bold we have the lightness and saturation numbers returned under the HSL colour model.

CLASS COLOUR HUE LIGHTNESS SATURATION RGB
1 61 85 / 100 100 / 30 rgb(255,255,178)
2 44 73 / 100 99 / 54 rgb(254,217,118)
3 34 65 / 100 98 / 70 rgb(254,178,76)
4 25 62 / 99 98 / 76 rgb(253,141,60)
5 10 58 / 99 97 / 83 rgb(252,78,42)
6 0 50 / 89 79 / 88 rgb(227,26,28)
7 347 35 / 69 100 / 100 rgb(177,0,38)

So we can see a gentle reduction in lightness - though the first five colours are all very light and pretty much indistinguishable in their lightness - but a steady increase in saturation.

Ordinarily we would use a single colour model to describe the colours in a graphic - HSL or HSV. But it does seem to me that it helps us detect variations in these colour components in meaningful ways if we ...

  • use HSL to detect changes in lightness across a colour scheme
  • use HSV to detect changes in saturation across a colour scheme

This will give you a sense of how these colour components vary as colour is applied to data graphics.

Qualitative Schemes

Let's briefly consider qualitative schemes in contrast.

These colour schemes, such as the ColorBrewer Accent7 scheme hues vary across the spectrum and not in an order that we can detect.

This explains why they work for nominal data, data that show difference, but that do not have a sequence.

Here are the descriptions of the colours in ColorBrewer Accent7 in HSL.
(HSV numbers are provided too in the lighter font, just to show that there is no consistent pattern here either - just variation, but we will focus on HSL below.)

Let's look at how value and saturation vary on the graph!
The colours in the ColorBrewer Accent7 seven class scheme go from left (1, green) to right (7, brown) here.

We can see that there is no consistent ordered variation in lightness and saturation. They are all over the place and vary a fair amount.

Here's what the wheel looks like with the hues in ColorBrewer Accent7.

The hues are well spread across the range to make them distinguishable, other than the colours for classes 3 and 7, which use similar hues (29 and 25) but vary lightness (76 and 47) to ensure that the colours are differentiable.

There is a risk that the reader will associate these colours as they may detect the similarity of hue.

Here we show the darker colours, where lightness values are lower, towards the centre of the wheel.

Adding rings that show variations in lightness (according to HSL) for each hue demonstrates how a single hue can produce different colours as lightness is varied. It confirms the variations in lightness as the colours that we see in these wheels are close to the colours used in the ColorBrewer Accent7 scheme - the colours that use a very similar hue.

This enables us to differentiate the colours from classes 3 and 7 used in ColorBrewer Accent7.

Note that the colours we see in the rings are not quite the same as those in the colour scheme. Can you explain why? Well ... there is one other component of colour that changes in the scheme, and here we are only varying hue and lightness!

I mentioned order above, and we saw how the ColorBrewer YlOrRd seven class scheme varies two of the components of colour in an ordered sequence.

Well, let's see how the colours of ColorBrewer Accent7 are ordered in the wheel.

You can see that the line that links the colours in order moves all over the place. There is no sequence across this colour wheel and so we see no sequence in the colours of the scheme. Remember that hue is not regarded as a visual variable that is ordered, so we don't usually use it to represent ordinal data.

But, as ever, there are exceptions - see ColorBrewer YlOrRd above!
